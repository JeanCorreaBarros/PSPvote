💡 FUNCIONES NUEVAS
═════════════════════════════════════════════════════════════════════════════

getRoleFromToken(): string | null
├─ Obtiene el token de localStorage
├─ Decodifica el JWT
├─ Extrae el campo 'role'
└─ Retorna el rol o null si no encuentra

decodeToken(token: string): TokenPayload | null
├─ Recibe un token JWT
├─ Lo divide en sus 3 partes
├─ Decodifica el payload
└─ Retorna el payload completo

⚠️ NOTAS IMPORTANTES
═════════════════════════════════════════════════════════════════════════════

1. TOKEN OBTENIDO DEL BACKEND:
   • El rol viene en el JWT
   • Campo: "role"
   • Valores soportados: ADMIN, LIDER (extensible)

2. DECODIFICACIÓN:
   • Sin validación de firma (confiamos en el servidor)
   • Decodificación manual sin librerías
   • Segura para uso en frontend

3. VALIDACIÓN REAL EN BACKEND:
   • Frontend solo controla UI
   • Backend debe validar permisos en CADA request
   • No se puede bypassear desde frontend

4. SINCRONIZACIÓN:
   • El rol se obtiene cada vez que se carga el dashboard
   • Si cambia el token, se recarga la interfaz

5. LOGOUT:
   • Limpia token y datos del usuario
   • Redirige a login
   • localStorage queda vacío
